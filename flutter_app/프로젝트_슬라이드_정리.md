# 프로젝트 발표 슬라이드 구성안

> 총 18~20장 기준, 각 슬라이드에 포함할 핵심 메시지를 정리했습니다.

---

## 1. 표지
- 프로젝트명: 건양대학교병원 환자 모바일 서비스
- 팀명/발표자 정보
- 앱 스크린샷 또는 로고 배치

## 2. 목차
- 배경 & 문제 정의
- 목표 및 기대효과
- 아키텍처 & 기술 스택
- 주요 기능 데모
- 지도·네비게이션 특화
- CI/CD & 배포 전략
- 문제 해결 & 개선 기록
- 향후 계획

## 3-4. 배경 & 문제 정의
- 기존 진료 예약/안내의 불편 사항 (전화, 대면)
- 환자/보호자 요구: 모바일 접근성, 실시간 안내
- 건양대학교병원 환경 & 데이터 (환자 수, 서비스 범위)
- “로그인 기반 개인화 + 모바일 지도 안내”의 필요성

## 5. 목표 및 기대효과
- 환자용 통합 앱 제공 (예약·알림·고객센터·지도)
- Django API + Flutter 앱 통합, 실시간 일정 공유
- CI/CD 자동화로 빠른 배포 → 사용자 경험 향상
- 병원 입장: 상담센터 부담 감소, 사용자 데이터 기반 분석

## 6-7. 전체 아키텍처
- 다이어그램 삽입 (Flutter ↔ Django ↔ MySQL ↔ React/Flask)
- 데이터 흐름 설명 (로그인, 예약, 지도 API 호출)
- AppState, Repository 패턴, Django REST ViewSet 역할

## 8. 기술 스택
- 표/아이콘으로 모바일·백엔드·배포·지도·DB 정리
- Flutter 3.27 / Django REST / MySQL / Kakao Map / GitHub Actions / AltStore

## 9-13. 주요 기능 데모
1. **로그인 & 환자 정보 로딩**
2. **진료 예약 UX** (시간 선택, 로그인 가드, 예약 생성)
3. **건강 알림** (로그인 상태별 변동)
4. **고객센터** (고객의 소리, 전화번호, FAQ)
5. **다가오는 일정 & 메인 화면** (동적 인사말, MY 탭)
   - 각 슬라이드에 UI 캡처 + 핵심 포인트 bullet

## 14. Kakao 지도 & 네비게이션
- 약국 안내 흐름: 현재 위치 → API → 최근 약국 → 경로 URL
- 병원 길찾기: 목적지 고정(건양대학교병원) + 출발지 자동 설정
- KakaoMap URL 파라미터 구조 설명 (sp/sn/ep/en/by)

## 15. CI/CD & 배포
- Git push → GitHub Actions → Flutter build apk/ipa → GitHub Releases
- React 웹 다운로드 버튼 + Django `latest-apk` API
- AltStore 설치/업데이트 절차 요약

## 16. 문제 해결 사례
- 예약 API 필드 불일치 → Flutter/Django 수정
- KakaoMap 목적지 미표시 → URL 파라미터 보강
- 마커 이미지 깨짐 → 기본 마커 + 이모지
- CI analyze 실패 → 옵션 변경
- 표 형태로 요약

## 17. 성과 & 지표
- 빌드 버전 히스토리 (1.0.5 ~ 1.0.10)
- 예약/알림/지도 기능 테스트 결과
- 사용자 흐름 데모 영상/캡처

## 18. 향후 계획
- Firebase Push 알림
- 챗봇(Flask 모델) 연계
- 자동 업데이트 안내, 관리자 대시보드 통합
- 서비스 고도화 일정

## 19. Q&A
- 질의응답 준비 슬라이드

### 활용 팁
- 각 슬라이드에는 “문제 → 해결 → 결과” 구조 유지
